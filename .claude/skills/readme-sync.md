# 更新根 README Skill

## 触发条件

当用户要求修改项目根目录的 README 文件时触发，包括：

- 修改 `README.md`、`README_zh.md`、`README_en.md` 中的任意一个
- "更新 README"、"修改根说明文档"
- 对仓库组织原则、贡献规范、目录结构等全局性说明的增删改

典型触发词：**README**、**根说明**、**组织原则**、**贡献规范**、**更新说明** + 仓库级文档。

## 核心规则

**修改 README 时，必须同时更新中英两份文件：**

- `README_zh.md` — 中文版
- `README_en.md` — 英文版

两份文件在结构和内容上必须保持同步。

## 工作流

### 1. 同时读取两份文件

```bash
Read README_zh.md
Read README_en.md
```

确认两份文件的当前状态，了解需要修改的段落在两份文件中的对应位置。

### 2. 按中文优先原则修改

- 先在 `README_zh.md` 中完成中文版本的修改，与用户确认内容无误
- 确认后，将相同改动翻译为英文，同步到 `README_en.md`

### 3. 翻译要求

- **正式文档风格**：使用正式、规范的书面语，避免口语化表达
- **术语一致**：仓库内已有术语保持一致，如 `Essential/` → "必装"/"must-have"，`Optional/` → "选装"/"elective"
- **示例同步**：如果中文版更改了文件路径示例（如 `Distrobox容器软件.md`），英文版必须使用相同的示例
- **格式对齐**：Markdown 结构（标题层级、列表类型、代码块类型等）必须与中文版完全一致

### 4. 变更前后对比

修改完成后，向用户展示两份文件的改动摘要，格式如下：

| 段落 | 中文变更 | 英文变更 |
|------|---------|---------|
| 组织原则 | 新增 xxx 条目 | Added xxx entry |
| ... | ... | ... |

确保所有变更都有对应的英文翻译，没有遗漏。

## 注意事项

- 路径分隔符统一使用正斜杠 `/`
- 平台目录名保持原始大小写（`Windows/`、`Linux/`、`Mac/`、`General/`）
- 如用户只要求改一份（如"更新 README_zh"），主动提醒并执行中英文同步修改
- 中文版的示例尽量使用仓库中真实存在的文件路径
