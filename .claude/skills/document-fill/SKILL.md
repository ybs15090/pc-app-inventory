# 补全软件文档 Skill

将空壳软件文档扩充为符合本仓库规范的完整文档。**务必先读仓库根目录 `CLAUDE.md` 确认最新规范**，本文档为工作流索引。

## 工作流

### 1. 读取目标文件

读取用户指定的 `.md`，保留其已有内容（TOC 结构、已有链接等），只在其基础上扩充，绝不推翻重写。

### 2. 读取规范与上下文

- 仓库根 `CLAUDE.md` — 文档规范（结构、命名、图标用法）
- 目标文件同目录的 `README.md` — 该文件夹定位与软件清单

### 3. 选风格参照

在同级目录找一份已完成的优秀文档模仿其风格，例如：

- `General/Essential/file-search/ripgrep.md`（单软件 + 表格速查）
- `Linux/Essential/效率软件.md`（一个文件对比多款软件）
- `General/Essential/VScode/vs-code.md`（软件 + 附属配置文件）

### 4. 检查媒体资源

在 `.media/svg/`、`.media/png/` 中搜索该软件图标。有则按 `CLAUDE.md` 的图标用法嵌入标题（`<img>` 标签，相对路径从目标文件位置计算）。

### 5. 研究软件（按需）

对该软件了解不足时，用 `WebSearch` / `WebFetch` 获取官网、安装方式、使用示例等最新信息。**不要凭记忆编造安装命令或版本号。**

### 6. 撰写文档

结构如下（与 `CLAUDE.md` 一致）：

```markdown
<!-- software-count: N -->   ← 仅在需要时加，见「软件计数」一节
# 目录 <!-- omit in toc -->
- [标题](#标题)
  - [安装](#安装)
  - [快速入门/使用方法](#快速入门使用方法)
  - [相关链接](#相关链接)

# 标题（含图标，如有）

一句话简介。

## 安装

（按平台分小节：Windows / macOS / Linux）

## 快速入门 / 使用方法

（核心功能，表格与代码块优先，便于速查）

## 相关链接

- 官网
- GitHub
- 文档/教程
```

**安装章节 — 每个平台同时给出两种方式：**

| 平台 | 包管理器（首选） | 本地安装包（备选） |
| --- | --- | --- |
| Windows | winget / Scoop / Chocolatey | `.msi` / `.exe` |
| macOS | Homebrew | `.dmg` |
| Linux | apt / dnf / pacman | `.deb` / `.rpm`、官网或 GitHub Releases 二进制 |

> Linux 平台始终附带 `.deb` / `.rpm` 等本地安装包方案 —— 部分场景用户无法或不愿使用外部包管理器。

**写作规范：**

- 中文撰写（`README_en.md` 除外）
- 仅限特定平台的软件，在简介中明确标注，并链接跨平台替代方案
- 命令速查、语法对照优先用表格；命令示例用 ` ```bash ` 代码块
- 破坏性操作（删除、格式化等）附 `>` 引用块风险提示

### 7. 软件计数（关键）

`scripts/count-software.py` 统计各平台软件数并写入 README 统计表，计数规则按优先级：

1. `<!-- software-count: N -->`（文件前 20 行内）—— 手动指定，最高优先级
2. `## <img` 开头的二级标题数量 —— 多软件文件
3. `# <img` 开头的一级标题数量 —— 单/多软件文件
4. 兜底 → 1

**何时必须手动加 `<!-- software-count: N -->`：**

- 一个文件收录多款软件，但**并非每款都有图标标题**（如 GParted 有 `# <img`、ncdu 只有纯 `# ncdu`）—— 自动计数会漏算，须在文件第一行写明真实数量
- 任何自动规则会误判数量的情形

单软件、或每款软件都带图标标题的多软件文件，无需手动加，交给自动统计即可。

### 8. 收尾

1. 更新同目录 `README.md`：把本文件加入清单（若未列出），附一句话用途说明
2. 运行 `python scripts/count-software.py` 刷新根 README 统计表
3. 向用户简要汇报新增的章节板块

## 注意事项

- 平台目录：`Windows/`、`Linux/`、`MacOS/`、`General/`（跨平台）、`Android/`、`iOS/`
- 非跨平台软件放对应平台目录，不放 `General/`
- 用户若移动过文件（平台变更），同步更新相关 README 索引与交叉引用的相对路径
