# IBus 中文输入法 — 常见故障与解决

收录 Ubuntu 下使用 `ibus-libpinyin` 时的常见问题及对应解决方案。

---

## 问题 1：输入延迟 / 卡顿，文字突然涌出

**现象：** 输入拼音后无响应，等待一段时间后文字突然全部涌出，不定时频繁发生。

该问题并非个例，详见 [GitHub Issue #429](https://github.com/libpinyin/ibus-libpinyin/issues/429)。

**临时解决方法：** 重启 IBus。

提供两个重启方式（见同目录附属文件）：

- **命令行脚本** [`ibus_restart.sh`](ibus_restart.sh)：可绑定系统快捷键，卡顿时按一下即可恢复。

  ```bash
  ibus restart
  ```

- **桌面快捷方式** [`Ibus_Restart.desktop`](Ibus_Restart.desktop)：键盘完全卡死时用鼠标点击触发。使用前右键 → **允许启动**。

**重装 IBus（可能根治）：**

```bash
sudo apt purge ibus
sudo apt install ibus ibus-libpinyin -y
```

## 问题 2：登录后输入法未自动启动

**现象：** 重新登录后无法切换到中文输入，状态栏无 IBus 图标，需手动运行才能恢复。

**原因：** IBus 守护进程未加入开机自启。

**解决方法：**

方法一：通过**启动应用程序**（Startup Applications）添加：

- 打开「启动应用程序首选项」→ **添加**
- 命令填写：`ibus-daemon --daemonize --replace --xim`

方法二：在 `~/.profile` 或 `~/.xprofile` 中添加：

```bash
export GTK_IM_MODULE=ibus
export QT_IM_MODULE=ibus
export XMODIFIERS=@im=ibus
ibus-daemon --daemonize --replace --xim &
```

> 修改 `~/.xprofile` 后需**注销重新登录**才能生效。

## 问题 3：Electron 类应用无法输入中文

**现象：** VS Code、Typora、Obsidian 等 Electron 应用中切换输入法后无反应，候选词框不弹出。

**原因：** 部分 Electron 应用未正确读取系统输入法环境变量。

**解决方法：**

在 `~/.bashrc`（终端启动的程序）或 `~/.profile` / `~/.xprofile`（图形界面登录）中添加：

```bash
export GTK_IM_MODULE=ibus
export QT_IM_MODULE=ibus
export XMODIFIERS=@im=ibus
```

修改后重启应用（若改的是 `~/.xprofile` 则需注销重登）。

**VS Code 专项：** 若上述方法无效，可在 `/usr/share/applications/code.desktop` 中的 `Exec=` 行追加启动参数：

```
--enable-features=UseOzonePlatform --ozone-platform=x11
```

## 问题 4：输入法图标从状态栏消失

**现象：** 任务栏/状态栏的 IBus 图标不见了，但输入法本身仍在工作（或不在）。

**解决方法：**

重启 IBus 守护进程：

```bash
ibus restart
# 或
pkill ibus-daemon && ibus-daemon --daemonize --replace --xim
```

若重启后图标恢复，建议检查是否存在多个 IBus 实例冲突：

```bash
ps aux | grep ibus
```
