>### 仓库文件格式规范

**组织原则：**
- 功能相关或相似的软件应放在**同一个文件夹**中，文件夹命名推荐以功能或用途命名，例如 `网络代理工具/`、`文本编辑器/`。
- 文件夹内每个软件的说明内容较少时，可以共用一个 Markdown 文件（如 `网络代理工具.md`）；如果某个软件内容较多，则建议为它单独创建一个 Markdown 文件（如 `Clash-Verge.md`）。
- 每个 Markdown 文件应该包含以下内容（推荐但不强制）：
  - 软件简介
  - 安装方法
  - 使用方法（简单说明即可）
  - 官方文档链接（如果有）
  - 其他相关资源链接（如教程、社区等）
- 如果要提供软件配置文件或其他相关资源，可以放在同名文件夹下，并在 Markdown 文件中提供链接。
- 所有内容应放在对应的平台目录下（Windows、Linux、Mac 或 General）。
